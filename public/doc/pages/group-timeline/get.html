リクエストURL

GET

パラメータ

必須パラメータ 値 説明
オプションパラメータ 値 説明

レスポンス(JSON)

[{
    "createdAt": [
        {number},
        {number},
        {number},
        {number},
        {number}
    ],
    "article" {string},
    "articleId" {string},
    "creator": {
        "name": {string},
        "groupId": {string},
        "icon": {string}
    },
    "likes": {number},
    "liked": {boolean},
    "comments": [
        {
            "commentatorName": {string},
            "commentatorId": {string},
            "commentatorIcon": {string},
            "comment": {string}
        }
    ]
}]
プロパティ名 値 説明
createdAt[] list タイムライン記事を作成した時間
createdAt[ 0 ] number 年
createdAt[ 1 ] number 月
createdAt[ 2 ] number 日
createdAt[ 3 ] number 時
createdAt[ 4 ] number 分
createdAt[ 5 ] number 秒
article string タイムライン記事
articleId string タイムライン記事のID
creator object 作成者の情報
creator.name string 作成者の名前
creator.groupId string 作成者のID（作成者自体がグループなので、groupId と表記してある）
creator.icon string 作成者のプロフィール画像
likes number いいね！の数
liked boolean いいね！をしているかどうかの判別
  • true - している
  • false - していない
comments[] list コメントのリスト
comments[].commentatorName string コメントしたユーザーまたはグループの名前
comments[].commentatorId string コメントしたユーザーまたはグループのID
comments[].commentatorIcon string コメントしたユーザーまたはグループのプロフィール画像
comments[].comment string コメント

複数のタイムライン記事に関して

複数のタイムライン記事を JSON に定義する場合、次のようにしてください。 

[{
    "createdAt": [
    ...
},
{
    "createdAt": [
    ...
},
{
    "createdAt": [
    ...
}]

また、最新の記事がリストの先頭に来るようにしてください。
